# Security policy

## Supported versions

Only the most recent release receives security updates. Static binaries do
not inherit fixes from the host operating system, so users should upgrade
when a new release is published.

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting for this repository. Do
not open a public issue for an undisclosed vulnerability. Include affected
versions, architecture, reproduction steps, and impact when possible.

Dependency vulnerabilities in X.Org or linked libraries are in scope when
they affect the pinned versions in `flake.lock`.
